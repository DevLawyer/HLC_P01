ABOUT HTML5

The World Wide Web (WWW) is a system that contains an almost infinite amount of information. But this Information must be arranged in some way so that it is possible to find what you are looking for. Unit Basic where this information is stored are the Web pages. These pages are characterized by containing text, images, animations ... and even sound and video. y.

One of the most important characteristics of Web pages is that they are hypertext. This means that Pages are not isolated elements, but are linked to others through links or hypertext links. Thanks to these links the internet navigator can click on a text (text that we will call from now on active) of a page to navigate to another page. It will be up to the home page programmer to decide which words or phrases will be active and where it will take us to click on them.





HTML ADVANTAGES


  • It allows you to develop applications that easily adapt to different resolutions, screen sizes, aspect ratios, and orientations.
  • It enables advanced capabilities such as GPS, camera and accelerometer to be enhanced on modern devices and provides a context-sensitive user experience on a wide variety of devices, including smartphones and electronic tablets.
  • Applications can be deployed as local web applications and can also be viewed in browsers.
  • Mobile apps can use the same monetization and distribution channels as native apps.
  • Applications are not restricted by browser window frames and can be run in full screen mode.
  • Users have complete control of devices and use of screen space.
  • The centralized code can be modified to interface with multiple devices.
  • JavaScript *, HTML, and CSS are the backbone of the Internet and web applications; therefore, the migration of development tools to mobile devices is easier. Writing integration code only once enables mobile applications to function similarly on all platforms, regardless of device.

HTML DISADVANTAGES


  • Sometimes mobile apps take time to get ready because developers have to write code for each platform.
  • Developers work in languages ​​that are not native to the platforms; therefore, the effectiveness of the code depends on the translation engine. It is often the case that coding techniques are inefficient or that the code is redundant.
  • The difference in platform runtimes often causes complications in cross-platform mobile application development.
  • The HTML5 standard evolved in a relatively short period of time, resulting in discrepancies in the implementation of CSS attributes, HTML tags, and JavaScript APIs. Sometimes these features behave differently on different platforms. However, there are tools available to resolve these discrepancies.

HTML FEATURES

  • The Web had to be distributed: The information spread over not very large pages linked together.
  • The Web had to be hypertext and it had to be easy to navigate.
  • It had to be compatible with all types of computers (PCs, Macintosh, workstations ...) and with all type of operating systems (Windows, MS-DOS, UNIX, MAC-OS, ...).
  • It had to be dynamic: the process of changing and updating information had to be agile and fast.

THE BASICS OF HTML

Once we have reviewed some basic notions of HTML and Web page programming in general, we will enter fully into programming with this language. We will soon realize the simplicity of this language which It makes it a very easy language to learn and that will allow us to create pages even more easily. The commands of this language will be made up of commands called labels that can have either the following structure: <LABEL_NAME> Or this other: <LABEL_NAME> TEXT </ LABEL NAME> As we can see, the first structure is formed by a single instruction and the second by two: one that marks the beginning of the label and another that marks the end, with text between them. For example, to tell the browser that we want it to paint a horizontal line we must write: <hr> As simple as that. As we can see, the name of the tag is delimited by the symbols less than (<) and greater than (>), all HTML instructions must be enclosed between these two symbols. But the labels are not limited to indicating such simple commands, these commands sometimes have parameters. For example, the <hr> tag as we have done previously would give rise to the line that we see in the figure 1.5, ie tells the browser to draw a horizontal line on the screen. This line has a thickness default and variable width depending on the size of the browser window.

GET STARTED WITH HTML

First of all we must specify that the text file we are writing is an HTML document, for this we use the start and end instructions of the tag at the beginning and end of the page respectively: <html> Page code: </html> The page code is in turn made up of two large blocks, the header and the body. The head of the page is delimited by the start and end instructions of the head tag. Are Instructions must be inside the HTML tag as follows: <html> <head> Header elements </head> ... Rest of the page code ... </html> All the information that will affect the entire page is entered at the top of the page. Initially this information will be limited to the title. This title will be indicated with the title tag using the following syntax: <html> <head> <title>My first WEB page</title> </head> </html> The result of viewing this page with our browser is the one shown in figure 2.2. As we see, the screen area where the page is usually is still empty, but if you look at the window header from the browser we see that the title has changed from being Microsoft Internet Explorer ™ to "My first WEB page - Microsoft Internet Explorer ™ ". In other words, the title that we give to our page with the <TITLE> tag will pass to be the title of the browser window.

HTML works with tags (tags) that allow us to organize the content of a web page. That is, thanks to a label we can indicate that a text, an image, a form, a table, a link, etc. will be displayed. The name of the tag is always written between the symbols . For example: <html> In addition, tags are usually represented in pairs: <html> </html> The first tag is called the start tag and the second end tag and within both we find the content. Element is dominated by the set formed by the start tag, the content and the end tag: <tag>content</tag> Then we have the attributes that provide extra information and will always appear included within the start tag. <atribute tag = "value"> As you can see the value of the attribute we will write it between double quotes. In short, we have the labels or tags; They can have attributes and the tags along with their content are called elements.

NOTE: HTML Tags are always written in lowercase letters

Some HTML tags are not closed, for example br and hr. <br> Tag: br stands for break line, it breaks the line of the code. <hr> Tag: hr stands for Horizontal Rule. This tag is used to put a line across the webpage.
